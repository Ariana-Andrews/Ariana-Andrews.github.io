CSS Positions

And How They Relate To HTML Elements

Static

There are five position styles in CSS, they are: static, relative, absolute, fixed and sticky. Static is the default state of html elements and is considered non-positioned, whilst the rest are considered positioned. When elements are in a static state,they sit on the page in the order they are written in html. 


           
<div class=”container”>
<div class=”block-1”></div>
<div class=”block-2”></div>
<div class=”block-3”><;/div>
</div>

Its important to note, that when elements are static, top, right, bottom and left styles do not apply to the element.

Relative

Relative is one of our ‘positioned styles’. It allows you to move the HTML element around with top, right, bottom and left styles. An element that has the position of relative will stay within the normal flow of the html document and will not impact the position of the other elements around it. When you apply top, right, bottom and left styles to a relative element, it will move in relation to its original position.

Absolute

The absolute position removes the element from the natural flow of the HTML document. In this example, if we gave block-2 the position of absolute, it would move and block-3 would take its place. One thing to note with absolute, is that the element will look for its nearest ‘relative’. If there are no elements in the mark-up with the position of relative, then it will move (with top, right, bottom and left styles) in respect to the view port.

Fixed

The fixed position is very similar to the absolute position, except it will only move in respect to the view port. This can be very useful for things like navigation bars, if you’d like it to always be visible when the user scrolls down the page.

Sticky

Sticky is a little more funky. If its parent box is a scrolling container, the element with the position sticky can switch between a relative position and an absolute/ fixed variant. When you give the element a top, right, bottom or left style, it tells the element when to ‘stick’ to the screen. Say you give it a top of 10px, once you have scrolled down far enough, the element will pop out of its usual position in the mark up and stay ‘fixed’ 10px from the top of the parent container.
